# AGENTS.md

本文件用于约束自动化协作者和维护者在本仓库中的操作方式，目标是减少误改、漏改和由上下文混乱造成的 bug。

## 20 条操作规则

1. 收到需求后，先判断需求类型和影响范围（文档、配置、资源、逻辑、bug、打包、架构等），阅读对应目录的 README、相关配置和必要源码；不要为了"了解项目"而全局扫描全部文件。阅读完成后，先给出执行方案（含假设、边界、影响目录），不修改任何文件；用户确认后再开始操作。
2. 修改前明确假设、边界和可能影响的目录；如果需求存在多种实施方案，逐一列出每个方案的优缺点和适用场景，不要采取"折中"或"混合"方案，避免弄乱原有结构；由用户选择后执行。
3. 新功能、主进程能力、IPC、打包、资源链路、公开 API 或架构调整属于高风险改动，必须先说明执行深度、验证方式和回退边界，不得直接修改。
4. 当需求涉及 bug 修复，或同一 bug 已修复多次仍未解决时，必须沿完整链路逐层排查根因，从源头消除问题；禁止用防卫式代码掩盖症状。
5. 新增或修改功能时，按可维护、稳定、可验证的产品级方案实现：职责边界清晰、状态归属唯一、错误路径可解释、接口命名见名知意，并补充与风险匹配的测试。
6. `electron-app/electron/main.cjs` 应保持为主进程装配、生命周期、依赖注入和薄包装入口；新增业务能力优先放入 `core/`、`pet/`、`behavior/`、`windows/`、`platform/`、`ipc/` 等对应职责目录。
7. 如果新需求不符合现有脚本或目录职责，不要硬塞进相近文件；应新建命名清晰、职责单一的目录或脚本，并同步补充对应 README 或 `docs/` 说明。
8. 发现相似逻辑时，先判断它是否属于同一种能力边界；若是，应接入已有接口或扩展现有模块，若只是表面相似，不要为了复用硬抽公共函数，也不要复制一份近似逻辑绕开既有边界。
9. 新增动画、音效、图标等运行资源时，必须确认本地运行、Windows release、Windows installer 以及适用的 macOS 打包链路都能携带该资源，并同步检查 manifest、variant profile、打包脚本和测试。
10. 只修改与当前需求直接相关的文件，不做顺手重构、格式化或风格迁移。
11. 优先沿用项目已有命名、脚本、目录结构和测试方式，避免引入不必要的新抽象。
12. 变更运行、构建、打包、依赖、资源生成、目录结构或公开 API 时，必须同步检查相关 README 和 `docs/` 文档。
13. 新增依赖、缓存、构建输出、临时目录或可再生成文件时，必须检查并按需更新 `.gitignore`，避免提交可再生成或本地专用内容。
14. AGENTS.md 面向自动化协作者和维护者，只写操作约束、阅读顺序、风险边界、验证要求和通用范围定位；任务特定指令放在对应目录的 README 或 `docs/` 中。
15. README 和 `docs/` 面向开源社区浏览者，说明项目、目录、脚本、功能和使用方式；不要写成交接记录、开发进度或 AI 提示。
16. 如确实需要临时交接或排查记录，放入 `.gitignore` 已忽略的位置，不纳入提交。
17. 修改逻辑代码时补充或更新与风险匹配的测试；只改文档时也要运行项目现有测试，除非依赖缺失或环境不支持，并说明原因。
18. 不提交密钥、用户私有路径、本机缓存、日志、依赖目录、运行时用户数据或打包产物。
19. 每次变更结束时给出清晰的验证结果、文档同步情况、是否更新 `.gitignore` 的说明，以及规范中文提交信息，建议格式为 `类型: 简短说明`。
20. 实施前用 `git diff --stat` 确认变更范围符合需求；如果变更文件数超出预期，暂停并向用户说明；完成后再次用 `git diff --stat` 核对最终变更范围。

## 常用范围定位

| 需求类型 | 优先阅读 |
| --- | --- |
| 项目入口、运行、打包、安装 | `README.md`、`electron-app/README.md`、`electron-app/package.json` |
| Electron 主进程、窗口、IPC、系统能力 | `docs/ARCHITECTURE.md`、`electron-app/electron/README.md` |
| 渲染层 UI、菜单、悬停面板、气泡 | `electron-app/static/README.md` |
| 宠物变体、动作资源、manifest | `assets/README.md`、`assets/animations/README.md`、`electron-app/electron/pet-variants.cjs` |
| 视频抽帧、抠像、动作替换、画质预览 | `tools/README.md` |
| 测试 | `electron-app/test/README.md`、`electron-app/package.json` |
| 文档同步规则 | `docs/MAINTENANCE.md`、`docs/PROJECT_MAP.md` |

## 变更后检查

- 运行与改动范围匹配的验证命令，默认至少执行 `cd electron-app; npm.cmd test`。
- 用 `rg` 检查被删除或改名文档是否仍被引用。
- 用 `git diff --stat` 确认变更范围符合需求。
- 若新增大体积或可再生成文件，确认 `.gitignore` 已覆盖。
- 最终回复中给出建议中文提交信息。
